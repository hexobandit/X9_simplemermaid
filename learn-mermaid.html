🚀 Master Mermaid.js Diagrams

Transform your ideas into beautiful diagrams with the simplest syntax you'll ever learn

What is Mermaid.js?

A powerful, text-based diagramming tool that creates professional diagrams using simple, Markdown-like syntax

📝 Text-Based

Write diagrams as code using intuitive syntax. No dragging, no clicking - just type and see your diagram appear.

⚡ Instant Rendering

Watch your diagrams come to life in real-time as you type. Perfect for rapid prototyping and quick iterations.

🎨 Professional Output

Generate beautiful, publication-ready diagrams that look great in documentation, presentations, and reports.

Getting Started

Learn the basics in just 5 minutes

🔰 Your First Diagram

Every Mermaid diagram starts with declaring its type. Here's the simplest flowchart:

flowchart TD A[Start] --> B[Process] B --> C[End]

This creates a top-down flowchart with three nodes connected by arrows.

Key Concepts: Try It Live →
  • 📍

    Direction

    TD = Top Down, LR = Left Right, BT = Bottom Top, RL = Right Left

  • 📦

    Node Syntax

    A[Text] creates a rectangle node with label "Text"

  • ➡️

    Connections

    --> creates an arrow between nodes

Diagram Types

Choose the right diagram for your needs

📊 Flowchart

Perfect for processes, workflows, and decision trees

flowchart LR Start --> Process Process --> Decision{Yes/No?} Decision -->|Yes| Success Decision -->|No| Retry

🔄 Sequence Diagram

Ideal for showing interactions between systems or components

sequenceDiagram User->>Server: Request Server->>Database: Query Database-->>Server: Data Server-->>User: Response

📅 Gantt Chart

Great for project timelines and task scheduling

gantt title Project Plan section Phase 1 Task A :done, a1, 2024-01-01, 7d Task B :active, a2, after a1, 5d section Phase 2 Task C :a3, after a2, 10d

🏗️ Class Diagram

Essential for object-oriented design and system architecture

classDiagram class User { +String name +String email +login() +logout() } class Admin { +manageUsers() } Admin --|> User

🔀 State Diagram

Show state transitions and system behavior

stateDiagram [*] --> Idle Idle --> Running : Start Running --> Paused : Pause Paused --> Running : Resume Running --> [*] : Complete

🗄️ Entity Relationship

Design database schemas and relationships

erDiagram CUSTOMER ||--o{ ORDER : places ORDER ||--|{ LINE-ITEM : contains PRODUCT ||--o{ LINE-ITEM : includes

Node Shapes & Styles

Express meaning through visual elements

🔷 Basic Shapes

Surround text with different brackets to create various shapes:

Shape Syntax Example Use Case
Rectangle A[Text] Standard box Processes, actions
Rounded A(Text) Soft edges Start/End points
Circle A((Text)) Perfect circle Key points, connectors
Diamond A{{Text}} Diamond shape Decisions, conditions
Hexagon A{{Text}} Six-sided Preparation steps
Database A[[Text]] Cylinder-like Data storage
Parallelogram A[/Text/] Slanted box Input/Output

Connections & Arrows

Show relationships and flow direction

Solid Arrow

Dotted Arrow

Thick Arrow

A ==> B

No Arrow

With Label

A -->|Label| B

Advanced Styling

Make your diagrams visually stunning

🎨 Custom Colors & Styles Try in Editor →

Use classDef to define custom styles and class or ::: to apply them:

flowchart TD A[Normal Node] B[Success]:::success C[Warning]:::warning D[Error]:::error classDef success fill:#5cb85c,stroke:#3e8e41,stroke-width:2px,color:#fff classDef warning fill:#f0ad4e,stroke:#ec971f,stroke-width:2px,color:#fff classDef error fill:#d9534f,stroke:#c9302c,stroke-width:2px,color:#fff A --> B A --> C A --> D
📦 Subgraphs for Organization

Group related nodes using subgraphs:

flowchart TB subgraph Frontend[Frontend Layer] UI[User Interface] UX[User Experience] end subgraph Backend[Backend Layer] API[API Server] DB[(Database)] end UI --> API API --> DB

Best Practices

Create professional, maintainable diagrams

  • ✅

    Use Descriptive Labels

    Instead of A, B, C, use meaningful names like User, Server, Database

  • ✅

    Break Complex Diagrams

    Split large diagrams into multiple smaller, focused ones

  • ✅

    Use Consistent Direction

    Maintain flow direction (TD for processes, LR for sequences)

  • ✅

    Apply Color Coding

    Use colors consistently (green for success, red for errors, etc.)

  • ✅

    Group Related Items

    Use subgraphs to organize related nodes visually

  • ✅

    Label Important Connections

    Add descriptions to arrows when the relationship isn't obvious
